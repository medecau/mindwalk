# mindwalk — impeccable

## Design Context

### Users
研究 agent 行为的工程师（首要用户是作者本人）。场景：在本机回放 Claude Code session，
观察模型对仓库的视野与遍历路径；平时长时间盯着分析，偶尔截图/录屏放进博客与研究报告。
信息密度和读数效率优先，但每一帧都要经得起截图。

### Brand Personality
安静、精密、有夜的诗意。三个词：**observatory / patient / luminous**。
情绪目标：像深夜在天文台看星图——专注、冷静，偶尔被一条轨迹击中。

### Aesthetic Direction
**夜城观测（Nocturnal Observatory）**。项目简报的备选名就是 lantern / firefly，
交互语言全是「微光 / 点亮 / 高亮」——这是一个关于光的产品，底色必须是夜。

- 深墨蓝夜空底（不是纯黑），仓库是黑暗中的确定性结构；模型触达哪里，哪里亮灯。
- 主视图是**萤火树**（放射状确定性树：目录为枝、文件为叶，萤火虫是主角，
  注意力深浅 = 地面光晕半径）；**注意力地形**（treemap 平原上隆起山脉）为切换视图。
  两者共享同一套光谱与回放语义。
- 光谱按触达等级：search = 磷光苔绿微光，read = 月白，edit = 暖钠灯琥珀，
  轨迹是萤火般的暖光线。**发光 = 数据，装饰永远不发光。**
- UI 是哑光仪器面板（slate 质感、细发丝线、小号大写字母眉题），不是玻璃拟态浮窗。
- 反面清单：霓虹青紫渐变、赛博朋克、glassmorphism、发光边框装饰、纯黑底。

### Type
- Latin 展示层用有个性的衬线/半衬线（自托管，随 dist 打包）；正文用人文主义无衬线。
- 中文回退 PingFang SC / Noto Sans SC，session 标题常为中文，混排必须精致：
  数字与单位用 tabular-nums，中英文之间留呼吸。

### Design Principles
1. **光即数据** — 亮度与色相只编码触达状态；任何发光的东西都意味着模型到过那里。
2. **仪器优先，画面成立** — 读数密度不妥协，但任何时刻截屏都是一张能发出去的图。
3. **夜而不赛博** — 光源是暖的（钠灯、月光、磷光），不是霓虹；夜空是墨蓝，不是纯黑。
4. **中英并重** — 字体栈与字号节奏为中英混排调校，不允许中文掉进糟糕的回退字体。
5. **动效克制** — 只在状态变化处用动效，指数缓出；完整响应 prefers-reduced-motion，
   回放推进提供无动画降级。
